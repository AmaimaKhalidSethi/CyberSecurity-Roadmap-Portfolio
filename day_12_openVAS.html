Vulnerability Scanning with OpenVAS / Greenbone

120-Day Cybersecurity Learning Roadmap • Docker Deployment Guide

Overview & Objectives

Today's focus covers the foundational concepts of vulnerability management, mapping out risk severity, and deploying automated tools to find known exposure points.

  • Concepts: Network vs. Vulnerability Scanning, CVEs, CVSS Scores, Lifecycle Management.
  • Target: Metasploitable2 (Host vulnerability simulation).
  • Tooling: Greenbone Community Containers (Docker).

Step-by-Step Docker Setup

1Prepare the Environment

Ensure you have Docker and Docker Compose installed on your host system. Create a dedicated directory for your Greenbone setup to keep data persistent and organized.

mkdir -p ~/cyber-lab/openvas
cd ~/cyber-lab/openvas

2Create the Docker Compose File

Greenbone provides an official community container stack. Create a file named docker-compose.yml using the highly stable, pre-packaged Greenbone Community Container setup:

version: '3.8'

services:
  gvm:
    image: greenbone/gvm-community:latest
    container_name: openvas_gvm
    ports:
      - "9392:9392" # Web UI HTTPS Port
    volumes:
      - gvm-data:/var/lib/gvm
    restart: always

volumes:
  gvm-data:
Note: The official Community Containers pull multiple microservices (gvmd, openvas-scanner, gsad). Using bundled community images like greenbone/gvm-community or securecodebox/operator simplifies local development significantly.

3Launch the Stack

Start the containers in detached mode. This step will automatically fetch the container images, provision the database, and spin up the web interface.

docker-compose up -d

4Update Vulnerability Databases (VT/SCAP Feeds)

OpenVAS relies on up-to-date vulnerability tests. Syncing the feeds can take anywhere from 10 to 30 minutes depending on your network connection. Run this inside the container:

docker exec -it openvas_gvm greenbone-feed-sync

To check the logs and monitor the syncing progress, use:

docker logs -f openvas_gvm

5Access the Web UI & Run Scans

Once the container is fully up and initialized, open your browser and navigate to:

https://localhost:9392

Default Credentials:

  • Username: admin
  • Password: admin (or check container creation logs for a randomly generated string)

From here, you can navigate to Scans > Tasks, target your Metasploitable2 IP instance, and analyze findings based on CVSS severity thresholds.
