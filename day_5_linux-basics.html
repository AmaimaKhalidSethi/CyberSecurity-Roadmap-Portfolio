Linux Basics II: Permissions, Users & Processes
Industry Insight 💡
Proper permissions and user management are the foundation of Linux security. In enterprise environments and penetration testing, misconfigured permissions (like writable /etc/passwd files or unprotected SUID binaries) are among the most common vectors for privilege escalation.

1. Linux File Permissions

Linux uses a UGO (User, Group, Others) model. Every file and directory is assigned three types of access rights: Read (r), Write (w), and Execute (x).

  • Read (r = 4): View file contents or list directory contents.
  • Write (w = 2): Modify file contents or create/delete files in a directory.
  • Execute (x = 1): Run a script/binary or enter a directory (cd).

2. User & Group Management

Managing who has access to the system is crucial. The root user has absolute power, while standard users operate in restricted environments.

# Create a new user named 'intern'
root@omnisec:~# useradd -m intern

# Set a password for the new user
root@omnisec:~# passwd intern

# Add user to the 'sudo' group for admin privileges
root@omnisec:~# usermod -aG sudo intern

# Switch to the new user account
root@omnisec:~# su - intern

Important Files:

  • /etc/passwd: Contains user account info (readable by all).
  • /etc/shadow: Contains secure user password hashes (readable only by root).
  • /etc/group: Contains group definitions.

3. Process Monitoring & Management

Every program running on Linux is a process assigned a unique PID (Process ID). Monitoring these helps detect malware, resource hogs, or unauthorized scripts.

# View currently running processes (snapshot)
user@omnisec:~$ ps aux

# View interactive, real-time process monitor
user@omnisec:~$ top

# Gracefully ask a process to terminate (PID 1337)
user@omnisec:~$ kill 1337

# Forcefully terminate an unresponsive or malicious process
user@omnisec:~$ kill -9 1337

Pro Tip: htop is a much more colorful, user-friendly alternative to top. If it's not installed, you can usually grab it via sudo apt install htop.
