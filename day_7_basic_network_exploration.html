Day 07: Foundational Mastery

Networking Fundamentals I: OSI Model & Network Exploration
Industry Insight 💡
Before you can analyze an attack, investigate a honeypot alert, or intercept credentials, you must understand exactly how that data moves. Malicious actors hide in the blind spots of network configurations. The OSI model isn't just theory—it is your diagnostic map for isolating where a breach occurred.

1. The OSI & TCP/IP Models

The Open Systems Interconnection (OSI) model breaks network communication down into 7 logical layers. While the real world primarily runs on the simpler 4-layer TCP/IP model, cybersecurity professionals use the OSI model as a universal language to describe where attacks happen (e.g., "This is a Layer 7 DDoS attack" vs. "This is a Layer 3 packet spoofing attack").

2. Network Exploration & Troubleshooting

A Blue Teamer must be able to quickly assess the health and topology of a network. These native Linux tools are your first line of reconnaissance and troubleshooting.

Testing Connectivity with ping

Uses ICMP (Internet Control Message Protocol) Echo Requests to determine if a host is alive and measure the round-trip time. It operates at Layer 3.

# Send exactly 3 pings to a Google DNS server
omnisec@soc:~$ ping -c 3 8.8.8.8
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=14.2 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=117 time=15.1 ms
64 bytes from 8.8.8.8: icmp_seq=3 ttl=117 time=14.8 ms

Tracing the Route with traceroute

Shows the exact path a packet takes across the internet to reach its destination by incrementally increasing the TTL (Time to Live) of packets.

# Trace the path to example.com
omnisec@soc:~$ traceroute example.com
1 router.local (192.168.1.1) 1.123 ms
2 isp-gateway.net (10.14.2.1) 12.450 ms
3 core-router-ny.net (203.0.113.5) 25.671 ms

Discovering Listening Services with ss

The modern, faster replacement for netstat. It dumps socket statistics, allowing you to see exactly which ports are open and listening on your machine. This is critical for finding unauthorized backdoor services.

# -t (TCP), -u (UDP), -l (Listening), -p (Show Process), -n (Numeric IPs/Ports)
root@soc:~# ss -tulpn
Netid State Recv-Q Send-Q Local Address:Port Peer Address:Port
tcp LISTEN 0 128 0.0.0.0:22 0.0.0.0:* users:(("sshd",pid=742,fd=3))
tcp LISTEN 0 511 0.0.0.0:80 0.0.0.0:* users:(("nginx",pid=815,fd=6))

Security Context: If you are monitoring a honeypot, running ss -tulpn verifies that your decoy services are properly bound and listening for attacker traffic.
