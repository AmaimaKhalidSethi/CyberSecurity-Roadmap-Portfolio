Why Nmap Matters

Before securing a system, you must understand what is exposed. Nmap helps security professionals:

  • Discover live hosts
  • Identify open ports
  • Determine running services
  • Detect service versions
  • Identify operating systems
  • Find potential vulnerabilities

In real-world engagements, Nmap is often the first tool used during reconnaissance.

What Is Port Scanning?

Port scanning is the process of checking which ports on a target system are open, closed, or filtered. Understanding open ports helps security professionals identify:

  • Running services
  • Potential attack surfaces
  • Misconfigurations
  • Security risks

Common Nmap Scan Types

Flag Scan Type Description Use Case Stealth Level
-sS TCP SYN Scan Half-open scan General reconnaissance High
-sT TCP Connect Scan Full TCP handshake No root privileges Low
-sV Version Detection Detect service versions Service enumeration Medium
-O OS Detection Detect operating system Host profiling Medium
-A Aggressive Scan OS, Version, Scripts, Traceroute Detailed reconnaissance Low
-p Port Selection Scan specific ports Targeted scanning N/A
--script NSE Scripts Run scripts Advanced enumeration Varies

Important Nmap Options

Skip Host Discovery

nmap -Pn <target>

Purpose: Assume the target is online; useful when ICMP is blocked.

Faster Scanning

nmap -T4 <target>

Purpose: Increase scanning speed; useful in lab environments.

Save Results

nmap -oN scan_results.txt <target>

Purpose: Save output to a file; useful for documentation and reporting.

Hands-On Lab (2–3 Hours)

Step 1 – Set Up Metasploitable2

Metasploitable2 is an intentionally vulnerable Linux machine designed for cybersecurity training.

  1. Download Metasploitable2.
  2. Extract the downloaded ZIP file.
  3. Open VirtualBox.
  4. Create a new virtual machine.
  5. Use the extracted VMDK file as the hard disk.
  6. Configure the network adapter as: Host-Only Adapter OR Internal Network.
  7. Start the VM.

Default Credentials: Username: msfadmin | Password: msfadmin

Step 2 – Identify Target IP Address

Inside Metasploitable2, run:

ip addr

Record the IP address (Example: 192.168.56.101).

Step 3 – Perform Basic Scan

nmap <target_IP>

Purpose: Discover open ports and identify basic services.

Step 4 – SYN Scan with Version Detection

sudo nmap -sS -sV <target_IP>

Purpose: Perform stealth scanning and detect service versions.

Step 5 – Aggressive Scan

sudo nmap -A -T4 <target_IP>

Purpose: Detect operating system, run default scripts, perform traceroute, and collect detailed information.

Step 6 – Vulnerability Script Scan

sudo nmap -sV -p 21,22,80,443 --script vuln <target_IP>

Purpose: Identify known vulnerabilities, enumerate services, and practice security assessment techniques.

Understanding Nmap Results

  • Open: A service is actively listening on that port.
    Example: 22/tcp open ssh
  • Closed: No service is listening.
    Example: 443/tcp closed https
  • Filtered: A firewall or filtering device is blocking access.
    Example: 80/tcp filtered http

Safety Guidelines

Only scan: Your Kali VM, Metasploitable2, systems you own, or authorized lab environments.

Never scan: Public IP addresses, company networks without permission, or unknown systems.

Unauthorized scanning may violate laws or policies.
